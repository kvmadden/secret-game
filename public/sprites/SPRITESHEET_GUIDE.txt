SPRITESHEET GUIDE — "On the Bench" Pharmacy Game
=================================================

This game supports drop-in PNG sprite sheets that automatically
override the programmatic pixel art. Place PNGs in this folder
and uncomment the matching entry in src/spritesheet.js.

GENERAL RULES:
- All sprites are TRUE 16x16 pixel art (no anti-aliasing)
- Use transparent PNG with alpha channel
- No padding between frames — pack tightly in a grid
- Pixel-perfect: every pixel should be intentional
- Color palette: 12-16 colors max per sheet for cohesion


FILE 1: pharmacist.png
-----------------------
Size: 96 x 64 pixels (6 cols x 4 rows of 16x16 frames)
Purpose: Main player character — the pharmacist

  Row 0: Walk RIGHT — 6 frames (standing, step1, mid-stride, step2, mid-stride2, step3)
  Row 1: Walk LEFT  — 6 frames (same sequence, mirrored)
  Row 2: Walk UP    — 6 frames
  Row 3: Walk DOWN  — 6 frames

Character description:
- Young pharmacy technician in white lab coat
- Name badge with green accent
- Dark pants, dark shoes
- Warm skin tone, short brown hair
- Friendly but focused expression
- The coat should have slight shadow/highlight distinction
- Frame 0 = idle/standing, frames 1-5 = walk cycle

OPTIONAL: Stress variants (expand to 96 x 256)
  Rows 0-3:  Stress level 0 (calm, normal colors)
  Rows 4-7:  Stress level 1 (slightly tense shoulders)
  Rows 8-11: Stress level 2 (red-tinted skin, hunched)
  Rows 12-15: Stress level 3 (very red, raised shoulders)

AI PROMPT for generating:
"16x16 pixel art sprite sheet, pharmacy technician character,
white lab coat, walk cycle 6 frames, 4 directions (right left up down),
Stardew Valley style, warm colors, transparent background, no anti-aliasing,
game asset, top-down RPG perspective"

WORKFLOW:
1. Generate with AI tool (PixelLab.ai or Stable Diffusion + pixel art LoRA)
2. Open in Aseprite or Piskel
3. Clean up to exact 16x16 grid alignment
4. Ensure transparent background (no white bg)
5. Reduce to 12-16 color palette per sheet
6. Export as PNG
7. Save to this folder (public/sprites/)
8. Uncomment matching entry in src/spritesheet.js
9. Refresh game — new sprites appear instantly, programmatic fallback gone


TESTING:
- Open browser console — you'll see:
  [SpriteSheet] Loaded 1/1 sprite sheets
- If a sheet fails to load, you'll see a warning and the
  programmatic sprites continue working as fallback
- No code changes needed beyond uncommenting the manifest entry
